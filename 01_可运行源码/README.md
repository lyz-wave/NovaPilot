# NovaPilot 3.0

AI 驱动的科研客户技术支持与咨询服务体系。当前实现对应 12 周“可信 MVP”的可运行纵向切片，覆盖：

- FFPE/RNA 项目事实与最小必要追问
- DV200、RNA 投入量与样本材料三重证据适用边界
- 循证检索结果和结论级证据绑定
- 科研 Reviewer、风险分级与科研决策卡
- 证据冲突强制转接和一次性交接包
- 专家认领、修订、批准与 SLA
- 科学建议、服务适配说明和授权式 CRM 分层
- 回答反馈、质量事件和 NovaBench 发布门禁
- 候选知识、Owner 审核、回归评测与门禁式灰度
- 客户咨询、专家工作台、知识进化、运营评测四个响应式页面

## 本地运行

```bash
npm install
npm run dev
```

打开 `http://localhost:3000`。

## 质量检查

```bash
npm test
npm run typecheck
npm run build
```

## 页面

| 页面 | 用途 |
| --- | --- |
| `/` | 科研咨询、项目事实、强制转接、科研决策卡 |
| `/expert` | 一次性交接、专家认领、冲突审查与正式批准 |
| `/knowledge` | 候选知识、Owner、NovaBench 与灰度治理 |
| `/operations` | 安全自助解决、质量事件与发布门禁 |

## 生产接入边界

仓库内的 SOP、SCI、CRM、工单、身份和模型调用均使用确定性适配数据，以便稳定演示和执行端到端测试；当前共享演示会话、进程内幂等记录不等同于生产认证与持久化。生产部署前需要接入诺禾 OIDC/SSO、租户级授权、持久化事件库、真实 CRM/工单接口、私有模型与数据策略，并用 300–600 条领域金标完成 NovaBench 冻结、影子评测和安全评审。
